# EventSheet AI - 60-Second Demo Script

## What To Show First

Open the app on `http://127.0.0.1:5173`.

Say:
"EventSheet AI is now based on a saved event record. The app is the live visual workspace, and the Excel Planning Book is generated from the same event record."

## 0-10 Seconds - Event Registry

Show:

- Saved events list
- `אירוע חדש`
- Last updated time

Say:
"Each event gets a stable eventId and is saved locally in this browser, so refresh no longer loses the event."

Click `אירוע חדש`.

## 10-25 Seconds - Intake And Organize

Show:

- Event name
- Client/family label
- Date
- Venue
- Estimated guest count
- Messy Hebrew text box

Click `טען דוגמת דמו`, then `ארגן מידע אירוע`.

Say:
"The manager can paste messy Hebrew event information exactly as it arrives. EventSheet AI organizes guests, suppliers, budget lines, risks, and seating notes into one event record."

## 25-40 Seconds - Live Workspaces

Show:

- Overview status cards
- Guest workspace with editable rows and dynamic category filters
- Supplier workspace with editable supplier rows

Say:
"This is no longer a temporary preview. Edits here update the saved event record that will generate the workbook."

## 40-52 Seconds - Seating Workspace

Click `הושבה`.

Show:

- Venue-style seating canvas
- Add table controls
- Table inventory
- Unassigned guests
- Click-based assignment flow
- Seating recommendation notes

Say:
"The seating screen is operational but still manager-led. It supports click-based assignment and recommendations, but it does not auto-seat guests. The manager decides."

## 52-60 Seconds - Workbook Continuity

Click `הפקה`, then generate/download the workbook.

Say:
"The Excel Planning Book is generated from this event record. Workbook links include the eventId, so clicking back to guests, suppliers, seating, or risks opens the same event on this browser."

## Out Of Scope

Say:
"This phase does not include database, login, CRM, WhatsApp, RSVP platform, payments, Office API, live Excel sync, or AI auto seating. Local persistence works on this browser/device; cross-device continuity requires a future backend."
